# Mishka Admin Backend

## Описание
Бэкенд панели управления. Отвечает за аутентификацию администраторов и управление динамической конфигурацией всех сервисов.

## Архитектура
*   **Inputs**:
    *   HTTP API (от Frontend)
*   **Outputs**:
    *   PostgreSQL (Таблица `dynamic_configs`)
    *   RabbitMQ (событие `config_updated` для оповещения сервисов)

## Переменные окружения (Static)
| Переменная | Описание |
| :--- | :--- |
| `ADMIN_PASSWORD` | Мастер-пароль для входа. |
| `JWT_SECRET` | Секрет подписи токенов. |
| `SUPERADMIN_ID` | Telegram ID главного админа. |
| `VIEWER_IDS` | Telegram IDs пользователей с правами просмотра. |
| `DEV_MODE` | `true`/`false`. Если `true`, позволяет вход без подписи Telegram (`initData="dev"`). |

## Динамические настройки (Dynamic)
Сам сервис не использует динамические настройки, но является их источником.
