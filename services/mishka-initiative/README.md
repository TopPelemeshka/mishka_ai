# Mishka Initiative

## Описание
Система проактивности. Анализирует каждое сообщение в групповых чатах и решает, должен ли бот вмешаться, если к нему не обращались напрямую.

## Архитектура
*   **Inputs**:
    *   RabbitMQ (`user_events`, Fanout): Слушает копию всех сообщений.
*   **Outputs**:
    *   RabbitMQ (`brain_tasks`): Если "интерес" высок, создает задачу для Мозга.

## Переменные окружения (Static)
| Переменная | Описание |
| :--- | :--- |
| `BOT_USERNAME` | Имя бота (без @) для проверки Hard Rules. |
| `MEMORY_API_URL` | URL сервиса памяти (для получения контекста). |

## Динамические настройки (Dynamic)
Управляются через **Admin Panel**.

| Ключ | Тип | Описание |
| :--- | :--- | :--- |
| `threshold` | int | Порог срабатывания (0-100). По умолчанию 70. |
| `aliases` | list | Список имен, на которые реагирует бот (Миш, Медведь...). |
| `soft_rule_instructions` | string | Промпт для LLM-судьи (критерии интереса). |
| `llm_model` | string | Модель для легкого "судейства" (обычно Flash). |
