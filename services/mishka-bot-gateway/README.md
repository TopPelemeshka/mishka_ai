# Mishka Bot Gateway

## Описание
Входная точка для взаимодействия с пользователями через Telegram. Получает обновления, обрабатывает медиа-файлы и пересылает события в систему через RabbitMQ.

## Архитектура
*   **Inputs**:
    *   Telegram API (Webhook/Long Polling)
    *   RabbitMQ (`bot_outbox`): Сообщения для отправки пользователю.
*   **Outputs**:
    *   RabbitMQ (`user_events`): Все входящие сообщения от пользователей.
    *   Файловая система (`/media`): Загруженные фото и голосовые сообщения.

## Переменные окружения (Static)
Следующие параметры задаются в `.env` и требуют перезапуска для изменения:

| Переменная | Описание |
| :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | Токен бота от BotFather. |
| `ALLOWED_GROUP_ID` | **ОБЯЗАТЕЛЬНО**. ID единственного чата, где бот имеет право работать. Сообщения из других чатов (включая ЛС) игнорируются. Бот **не запустится** без этой переменной. |

## Динамические настройки (Dynamic)
*Нет динамических настроек.* Сервис выполняет только транспортную функцию.
