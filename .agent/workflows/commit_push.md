---

## Steps

1.  **Check Status**
    - Выполни команду `git status`, чтобы увидеть измененные файлы.
    - Если изменений нет, сообщи пользователю и заверши работу.

2.  **Analyze Diff**
    - Выполни `git diff` (или `git diff --cached` если файлы уже в stage), чтобы понять суть изменений.
    - Проанализируй контекст изменений: это новая фича, исправление бага или рефакторинг?

3.  **Generate Message**
    - Придумай название коммита в формате Conventional Commits, но НА РУССКОМ ЯЗЫКЕ.
    - Примеры:
        - `feat: добавил микросервис для работы с календарем`
        - `fix: исправил ошибку подключения к RabbitMQ`
        - `docs: обновил документацию по запуску`
    - Сгенерируй также краткое описание (body) коммита, если изменений много.

4.  **User Approval**
    - Покажи пользователю предложенное сообщение коммита.
    - Спроси: "Согласны с таким сообщением? Отправляем?"

5.  **Execute**
    - Если пользователь согласен:
        - Выполни `git add .` (или конкретные файлы, если обсуждалось).
        - Выполни `git commit -m "ВАШЕ_СООБЩЕНИЕ"`.
        - Выполни `git push`.
    - Сообщи об успехе.
