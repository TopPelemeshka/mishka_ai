# Основные правила поведения

1.  **Язык общения:** Строго Русский язык. Все объяснения, мысли, планы и диалоги с пользователем — на русском.
2.  **Комментарии и документация:** Все Docstrings, комментарии в коде и README файлы пишем на русском языке.
3.  **Именование коммитов:** Используем Conventional Commits, но описание на русском (пример: `feat: добавил сервис генерации мемов`, `fix: исправил ошибку в Dockerfile`).
4.  **Стиль кода:** Python 3.11+. Обязательная типизация (Type Hints). Соблюдение PEP8. Использование асинхронности (`async/await`) везде, где это возможно.
5.  **Безопасность:** Никогда не хардкодить API ключи. Всегда использовать переменные окружения (`os.getenv`).
